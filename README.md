# 朋友圈社交平台项目
基于 Python Flask + MySQL + JS 实现的简易朋友圈功能，支持发布、修改、删除帖子，可见性控制（公开/仅好友/仅自己）、好友动态、评论等。**项目目前处于开发阶段，部分功能尚未完全实现。**

## 项目结构
```
├── /src # 数据库操作相关代码
│ ├── config.py # 数据库配置文件
│ ├── init_db.py # 数据库初始化脚本
│ ├── create_table.sql # 数据库建表文件
│ ├── create_trigger.sql # 数据库创建触发器
│ ├── user_service.py # 用户相关代码
│ ├── friend_service.py # 好友相关代码
│ ├── post_service.py # 朋友圈，评论相关代码
│ └── admin_service.py # 管理员相关代码
├── /app # 后端核心代码
│ ├── app.py # 后端主程序
│ ├── /app/templates # 前端 HTML 模板文件
│ ├── /app/static/js # 前端 JavaScript 脚本文件
├── run.py # 项目启动入口
```

## 功能介绍
### 👤 用户端功能
#### 已完成
1. 用户注册、登录，自动初始化用户 ID
2. 支持修改个人信息：姓名、性别、出生日期等
3. 好友体系：支持添加、删除好友
4. 朋友圈功能：发布、编辑、删除动态，可见性控制（公开/仅好友/仅自己），自带字数限制
5. 朋友圈自动记录最后更新时间
6. 可查看好友朋友圈、查看评论、发表评论
7. 删除朋友圈时，自动级联删除对应所有评论数据

#### 未完成
1. 好友申请确认
2. 通过用户名搜索用户

### 🛠️ 管理员端功能
#### 已完成
1. 独立管理员账号密码登录系统
2. 管理员可修改自身个人信息
3. 支持注销用户，一键清除该用户所有系统数据
4. 权限隔离：管理员无法查看用户隐私信息
5. 可查看全站所有朋友圈内容，支持审核、违规删除动态

## 技术栈
- 前端：HTML / JavaScript 
- 后端：python Flask（根据你实际的改）
- 数据库：MySQL

## 运行步骤
1. **配置数据库**
   - 打开 `src/config.py` 文件，将 `ROOT_DB_CONFIG` 中的数据库密码替换为你本地的 MySQL root 密码。
   - 配置完成后，运行 `src/init_db.py` 完成数据库初始化：
        ```bash
        python src/init_db.py
        ```

2. **启动项目**
    - 执行主程序文件：
        ```bash
        python run.py
        ```
3. **访问系统**
   - 打开浏览器，访问以下地址进入系统：http://localhost:8000/

4. **初始数据**
   - 初始存在管理员：id：1；密码：9876543210
   - 初始存在三个用户：
      1. 用户名：aaa；密码：aaaaaaaa；性别：male
      2. 用户名：bbb；密码：bbbbbbbb；性别：female
      3. 用户名：ccc；密码：cccccccc；性别：other
